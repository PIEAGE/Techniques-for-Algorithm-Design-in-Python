General


Order of Operations


  1. Try to solve the problem on your own
  2. Write down your attempt on paper
  3. Consider the following test cases
    1. General Cases
    2. Base Cases
    3. Error Cases

Approahing the Problem


  1. Listen Carefully
    Listen to the question carefully to extract important information for ex the phrase "sorted list "

  2. Upon drawing an example make sure it is:
    1. Specifc
    2. Sufficiently large
    3. Not a special case - We want a general case at first

  3. Brute force solution
    Write down brute force solution and discuss its complexity and then dive into improvements

  4. Optimise
    1. Did you use all the question information?
    2. Use a fresh example
    3. Use an imperfect solution to work out why it isn't working and what a perfect solution would do instead
    4. Use extra space to see if it helps run time
    5. Can we sort values or carry out some operation that will save us time in the long run?
    6. Use a hash table

  5. Walk through your code
    Solidify your understanding of what you've just written

  6. Implement
    Add error checking as a todo to your code

  7. Test your code
    Add error checking as a todo to your code

BUD Optimisations

Bottlenecks
Look for the worst complexity part and see if you can improve it - it's often searches or moving through datastructures which are the most inefficient parts of algorithms

Unnecessary Work
Try and remove multiple loops by looking at the question from a different perspective

Duplicated Work
Quite simply are we doing something more than once? Here is where we might be able to utilise a hash set

Other tips

Perception
Remember to not just think of these tasks as computer algorithms but approach them from the perspective of what you would do without a computer first! Often you'll find you intuitively know what to do and then can convert that intuitive process into code

Unnecessary Work
Try and remove multiple loops by looking at the question from a different perspective

Data Structures
Run through all the data structures you know about and see if they could be applied to the problem to gain better efficiency

Python Counter
 Code 

    from collections import Counter

    str1 = "abcdefgg"
    counter = Counter()
    for c in str1:
      counter[c] = counter[c] + 1
    print(counter)

 Output 

    Counter({'g': 2, 'a': 1, 'b': 1, 'c': 1, 'd': 1, 'e': 1, 'f': 1})

Head and Tail
 Code 

      current = ll.head
      while current.next:
        print(current.value)
        current = current.next



Explanation

This code is the same as before but notice that we are making use of the head. This is such an important part of interacting with LinkedLists so don't overlook it

Runners
 Code 

      #Assume Linked List 5 -> 4 -> 3 -> 2 -> 1

      curr = ll.head
      while curr:
        print('Curr:' + str(curr))
        runner = curr

        while runner.next:
          print('Runner:' + str(runner))




 Output

        Runner: 3

        Runner: 1
